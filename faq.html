Necessities

Who should I cite?

At the moment no one, and you're not permitted to edit any of the source code, or indeed to publish anything generated from or written in this package without the express permission of the author. Once the paper is out, however, all this will change.

What programs does this use? And how do I cite them?

Only cite the programs you use from the following list:

  • RAxML - 7.3.0
  • BEAST - 1.7.1
  • MUSCLE - 3.8.31
  • MAFFT - v6.847b
  • Clustal-Omega - 1.0.3
  • PRANK - v.100701
  • Phylomatic - 4.2 (part of Phylocom)
  • MetAl - 1.1 (compiled by me)
  • TrimAl - 1.2rev59

But you must cite BioPython. Sorry not to be more help than this, but it's best the authors tell you what to cite so I don't get in trouble!

phyloGenerator keeps hanging and not doing anything!

I've set phyloGenerator to pause every ten times it downloads something for ten seconds, so as not to overload the NCBI database. However, I'm being massively conservative, and if you want to alter that (at your own risk!) then play around with the 'delay' argument when running phyloGenerator. If in doubt, why hurry?

I keep getting warned about server errrors!

This is nothing to worry about, unless it crashes phyloGenerator or causes it to exit. If either of those things happen, consider not downloading as many species (more than six hundred seems to cause it problems, and I've not designed this program to handle that many), and please send me an email about it.

It doesn't work! Help!

Oh no! Contact me using the link at the top of this page ('Who?'), including a copy-pasted version of everything you did and got back from phyloGenerator, any files you gave to phyloGenerator, and details about your computer (Mac? Windows? What version?). I'll get back to you quickly, I promise.

Nine times out of ten, the problem is that you're working with sequences you should have trimmed, or the constraint tree you've specified is rooted and so RAxML has unexpectedly crashed. If you can't load file in, make sure you've specified the absolute path ('/Users/will/file.txt', not 'file.txt'), and that there're no spaces at the end of your input ('C:\Docs\file.txt', not 'C:\Docs\file.txt ').

I don't understand what this program does! Help!

Have a read through the walkthrough. If that doesn't help, drop me an email. I'm afraid I can't explain the whole of phylogenetics to you (sorry!) but I'm quite likely to help you if you send me a polite email. Perhaps an offer of a beer would also be appropriate :p

I don't think this program is a good idea. It doesn't include method X / philsophically it's a bad idea / etc.

I probably agree with you. Please, send me an email (the 'Who?' link at the top should help you find me) and let's talk about it. Maybe we can improve the program, but at the very least I'd appreciate your feedback. For the record, I certainly don't think this program is a replacement for phylogeneticists. Otherwise I'd be out of a job!
