Overview

phyloGenerator is simple enough to use, but it does take time to download everything from GenBank. Patience and a cup of tea are all that's required, however! Whenever you need to input a file, if it's not in the same folder as phyloGenerator you'll need to give phyloGenerator the 'absolute path' to the file. Something like '/Users/will/Documents/dna.fasta', or 'C:/Documents and Settings/dna.fasta'.

There are some example files included with phyloGenerator - to use them, simply type in whatever you see in italics wherever prompted below.

Using an options file will give you more control over phyloGenerator, and its use is described in the 'reference' section. However, you can also use it interactively, and this will be described below. I will assume that you have successfully installed phyloGenerator, and have started it running.

Start-Up

Stem Name

phyloGenerator will automatically output all your files, but you need to give it a 'stem name' so that it knows what to call everything. I'd suggest 'test' for your first attempt!

DNA Input

If you've already downloaded or sequenced your data, but not aligned it, tell phyloGenerator where that FASTA formatted file is now. Otherwise, just press enter.

An example file here is test.fasta.

DNA Download

If you don't have DNA sequences already, you'll need to download some. Tell phyloGenerator where a plain text file containing the binomial names of the species you want is. Each species must be on a separate line, with no formatting of any kind in the file. Otherwise, just press enter.

If you're downloading DNA, you'll need to provide a valid email address so Entrez can contact you about your downloads - don't worry, they won't if you provide a real one! You then need to give the genes you want to search for. If you want to give more than one, make sure you enter them on separate lines, and simply press enter on an empty line to continue once you're done.

You may have to wait some time before your sequences have all been downloaded, but phyloGenerator will tell you how it's doing. By default, it waits ten seconds after downloading each DNA sequence so that it doesn't irritate the servers at GenBank. You can change this behaviour in the options file, if you like.

An example file here is test.txt.

DNA Checking

You will only get to this stage if you have some DNA from the previous steps loaded in.

This table shows each DNA sequence's 'ID', name used when searching GenBank (if appropriate), the name it was given on GenBank (usually an ID number), and its length. Sequences that are in the top or bottom 10% of sequence lengths are highlighted in the list. If the sequence lengths are within the 'specified tolerance' (by default, the minimum sequence length should be no less than 90% of the maximum length) it will be stated, but you can continue with intolerable sequences.

There are three modes when 'checking' DNA: delete, reload and trim. You can enter any of the modes by typing their names. To continue, simply type continue.

delete
Enter a single sequence's 'Seq ID' to delete it.
reload
Enter a single sequence's 'Seq ID' to download another sequence from GenBank with the search term originally used. By default, phyloGenerator will look through more sequences and take the sequence with the length closest to median of the others. You can change these defaults in the parameter file.
trim
Enter a single sequence's 'Seq ID' to trim a sequence down to only its Open Reading Frame (ORF). If you want to trim all the sequences (and that's advisable), enter 'all' and press enter. To change the translation table for the sequence you're working with (it might be from a plant's chloroplast, for example), type translation and select the appropriate number for each gene. Note that phyloGenerator will likely have already trimmed a sequence originating from a whole plasmid (e.g., a mitochondrial gene), by extracting what has been annotated as that gene.

Alignment

Alignment

Simply press enter, and phyloGenerator will align your DNA using MAFFT. However, you have a choice of programs: MUSCLE, MAFFT, or Clustal-Omega. To use any of them, simply enter muscle, mafft or clustalo respectively. To compare the alignments of several programs, enter everything.

Alignment Checking

It is strongly recommended that you look at your alignment before using it. Summary statistics for your alignment(s) have been outputted, and you may notice that one alignment has more gaps than another, or that your alignment seems OK. You can write out your alingment(s) by typing output. There are several modes available to you now, depending on what you enter. There are three modes when 'checking' an alignment: edit, align and trim.

edit
This takes you back to the DNA editting stage. When you continue through it, you will skip straight to the alignment phase.
align
This will take you back to the alignment stage, so you align your sequences again, possiblly with different options.
trim
Some areas of your alignment may be filled with gaps, and some phylogeneticists woudl argue these areas should be removed, as they are uninformative. This mode uses 'trimAl' to automatically remove such regions, and will output an HTML file that describes what has been removed.

Phylogeny Generation

Constraint Tree

It is strongly advised that you use a constraint tree in your analysis. Generating novel phylogenetic hypotheses without a fair amount of knowledge is a tricky thing, so by constraining your tree to fit in with the general consensus of how your group fits together, you're reducingt the likelihood that your results will be untenable.

If you already have a constraint tree, type in its file name here. If you want to generate one from GenBank's taxonomy, type genbank taxonomy, or if you want to try the online PlantMiner database, type plantminer (only for plants). The latter will take some time, and requires an Internet connection.

An example file here is test_constraint.tre, but it can only be used with the FASTA file test.fasta. Otherwise, simply press enter to continue without a constraint tree.

RAxML

This is it! By simply pressing enter, you will make a phylogeny using RAxML default rapid boostrapping methods. To conduct more searches (advisable), simply type the number of times you would like to restart your search, and phyloGenerator will automatically select the best tree for you and annotate it with bootstrap values.

Rate Smoothing

RAxML creates phylogenies whose branch lengths are proportional to the number of mutations that have taken places along them. However, for most purposes, you will want a phylogeny whose branch lengths are proportional to time. To use PATHd8 to make such a rate-smoothed phylogeny, simply enter the name of your outgroup species and press enter. Otherwise, press enter to continue without making one.

End of the line!

You're now done! phyloGenerator has produced intermediate files (sequences downloaded, aligned sequences, a log of what was done) and your final phylogeny/phylogenies with the stem name you specified at the beginning. In the even that phyloGenerator crashes, it will attempt to write out everything it has collected so far, so hopefully nothing will be lost even if something goes wrong.
